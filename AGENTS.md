# Vocab Agent — Agent Instructions

Self-evolving English learning AI with dual-agent architecture (Teacher + Developer), generative UI, and FSRS spaced repetition.

## Commands

- `npm run dev -- --turbopack --port 3088` — dev server (Turbopack required; webpack is extremely slow on junction/Q: drives)
- `npm run build` — production build (webpack)
- `npm run lint` — ESLint (next/core-web-vitals + typescript)
- `npm test` — unit tests (vitest, `src/**/*.test.ts`, 30s timeout)
- `npm run test:e2e` — E2E tests (requires dev server running at `localhost:3088`, 120s timeout, set `E2E_BASE_URL` to override)
- `npm run db:migrate` — run DB migrations (`npx tsx src/lib/db/migrate.ts`)
- `npm run db:reset` — reset DB (`npx tsx src/lib/db/reset.ts`)
- `npm run import-ecdict` — import ECDICT dictionary data

## Architecture

**Dual-Agent routing** (`src/lib/ai/agent-router.ts`):
- `/dev` or `/d` prefix → Developer agent; regex patterns also detect dev intent
- Teacher patterns (e.g. "什么是X", "帮我复习") override dev patterns → Teacher
- Default → Teacher

**Chat API** (`src/app/api/chat/route.ts`):
- AI SDK v7 `streamText` with `convertToModelMessages` + `UIMessage` format
- `instructions` field (not `system` messages) — AI SDK v7 forbids system messages in `messages[]`
- Context manager trims old messages and appends summary to instructions
- Max 25 tool-call steps (`stepCountIs(MAX_STEPS)`)
- UIMessages are sanitized before conversion (orphaned tool parts from `/review`, `/stats` commands are stripped to avoid `AI_MissingToolResultsError`)

**Teacher agent** (`src/lib/ai/teacher-agent.ts`): fast model, tools: fsrs-review, fsrs-rate, vocab-lookup, add-word, extract-words, dict-lookup, vocab-stats, pin-word, unpin-word

**Developer agent** (`src/lib/ai/developer-agent.ts`): reasoning model, tools: file-write, file-read, file-edit, file-list, shell-exec, create-command, register-tool, register-component, unregister-component, db-query, save-lesson, list-lessons, merge-lessons, test-command

**Models** (`src/lib/ai/models.ts`):
- Teacher: `openaiCompatible.chat()` — uses `/chat/completions` endpoint (not `/responses`)
- Developer: `deepseek()` — reasoning model, returns `reasoningText` stream parts
- Configured via `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `TEACHER_MODEL`, `DEVELOPER_MODEL` env vars

**Command system** (`src/lib/commands/`):
- Built-in: `/review`, `/add`, `/stats`, `/rate` — executed without LLM
- Dynamic: stored in `dynamic_commands` DB table, executed in sandbox via `new Function()` with injected `db`, `tables`, `dql`, `fsrs`, `args`, `console`
- `/dev` and `/d` are NOT commands — they route to the Developer agent via the chat API

**Generative UI** (`src/components/generative/`):
- `component-registry.ts` is AUTO-GENERATED by register-component/create-command tools — DO NOT EDIT MANUALLY
- Components live in `src/components/generated/` — registered via static imports, hot-reloaded by Turbopack HMR
- Component name must match the `type` field returned by the tool/command

**World State** (`src/lib/pipeline/world-state.ts`): extracted via parallel extractors, injected into Teacher agent instructions as JSON context

**Database**: `@libsql/client` (WASM-based SQLite) + Drizzle ORM. DB file at `data/vocab.db`. Schema in `src/lib/db/schema.ts`. `better-sqlite3` does NOT work in this environment.

**Auth**: middleware (`src/middleware.ts`) checks `vocab-auth` cookie against `AUTH_PASSWORD` env var. All routes except `/login`, `/api/auth`, and static assets require auth.

## Gotchas

- **Turbopack required for dev** — `npm run dev -- --turbopack`. Webpack is unusably slow on this Windows setup.
- **`@libsql/client` not `better-sqlite3`** — native compilation fails; `serverExternalPackages: ['@libsql/client']` in next.config.ts is required.
- **AI SDK v7** — uses `instructions` (not system messages), `UIMessage.parts` (not `content` string), `convertToModelMessages()`, `stepCountIs()`. Read `node_modules/next/dist/docs/` and AI SDK docs before assuming v3/v4 patterns.
- **`component-registry.ts` is auto-generated** — editing it manually will be overwritten next time a component is registered.
- **`generated/` is gitignored** — agent-generated code (tools, scripts, components) lives here and is not tracked.
- **E2E tests need a running dev server** at `localhost:3088` (or set `E2E_BASE_URL`).
- **Node.js 20.x** — AI SDK may warn `EBADENGINE` (wants >=22), but it works. Ignore the warning.
- **DeepSeek reasoner** — returns `reasoningText` parts; `onStepFinish` may not fire, so debug logs use `onEnd` fallback.
- **Deploy** — `deploy.sh` targets Ubuntu/Debian VPS with Caddy + DuckDNS. Service runs on port 3088, Caddy proxies on 31588.
- **Path alias** — `@/*` maps to `./src/*` (configured in tsconfig.json and vitest configs).

# 自进化英语学习 AI Agent 系统设计方案

本方案融合了原始设计思路与自进化 Agent、Generative UI 的前沿架构建议，旨在打造一个能够通过聊天交互“动态生长”功能、兼具高实用性与极客感的英语学习系统。

---

## 1. 应用预期用途与创新性

### 1.1 核心用途
* **高效背词与应用：** 结合渐进式间隔重复算法（FSRS），帮助用户科学记忆单词并加深理解。
* **英文资料阅读辅助：** 辅助用户在阅读英文长文本、技术文档或小说时，进行生词提取、上下文分析和无缝翻译。

### 1.2 核心创新点：“软件自进化”
传统软件的功能是由开发者预先编码并配置好的，而本系统核心创新在于**“界面和功能的低代码/无代码自进化”**。
* 基于一个极简的 Web 聊天交互界面，用户可以通过自然语言指令，命令 Agent 动态编写代码来完善、扩展系统自身的功能。
* 最终，系统的大多数命令式交互（例如新的背词游戏、特定的统计图表、自定义的复习模式）都将由 Agent 自身生成的代码来实现并运行。

### 1.3 进化的三个维度
1. **表现层进化（Generative UI）：** 诸如单词卡片样式、倒计时组件、连线消除小游戏等，由 Agent 实时生成代码并在前端动态渲染。
2. **逻辑层进化（Dynamic Tools）：** 用户通过 `/` 开头触发的自定义功能，由 Agent 编写后端脚本，直接升级系统的技能库。
3. **数据层进化（Dynamic Extraction）：** 决定“Agent 能看到什么数据”的中间脚本层也是可进化的，系统会根据新功能自动调整数据聚合规则。

## 2. 基础架构设计

### 2.1 交互界面 (Interface & Generative UI)
* **自适应多端：** 基于 Web 的现代化聊天界面，完美适配 PC 端和手机端。
* **输入框多模态：** 支持正常的自然语言聊天文字，以及精简的 `/` 开头的命令模式。
* **高流利度单词卡片：** * 引入双面卡片设计（正面单字/音标，反面释义/例句/应用场景）。
  * 底部提供类似 Anki 的反馈按钮。全面支持键盘快捷键，卡片展示模式下聊天框自动失去焦点（Blur）。
* **Generative UI 落地机制：**
  * 依托 **Vercel AI SDK UI** 监听大模型输出的结构化 JSON。
  * 前端配置动态组件注册与沙盒渲染环境，确保 Agent 新写的 React 组件能无缝展现。

### 2.2 Agent 实现 (Dual-Agent 架构与数据流设计)
为了同时兼顾“英语教学”的专业性与“系统开发”的逻辑性，系统采用**双层 Agent 架构 (Dual-Agent Framework)**，并设计了动态的数据提取流。

```text
                       [ 用户交互界面 (UI) ]
                                |
             +------------------+------------------+
             |                                     |
    [ Teacher Agent ]                     [ Developer Agent ]
  (生词纠错、例句生成、FSRS计算)         (理解功能需求、编写/重构代码)
             |                                     |
             +------------------+------------------+
                                |
                     [ 全局世界状态 (World State) ]
                  (包含学习数据摘要、当前已有的 /commands)
                                |
                 **[ 动态数据提取层 (自进化脚本层) ]**
                                |
                       [ 底层数据库 (DB) ]
                    (全量 FSRS 记录、聊天日志等)
```

* **Teacher Agent（教学智囊）：** 负责日常交互、FSRS 算法调度。
* **Developer Agent（自进化引擎）：** 负责理解需求，编写代码，修改数据库 Schema 或动态调整 System Prompt。它不仅保存技能，还会编写调试代码来完成用户的定制要求。

### 2.3 核心亮点：自进化数据提取层 (Data Pipeline)
从底层数据库到注入 Agent 上下文的“World State”之间，必须存在一个**动态数据提取脚本层**，避免把整个数据库塞给大模型导致 Context 溢出。
* **按需降维与聚合：** 原始数据库里可能是上万条复习记录日志。提取层脚本（如特定的 SQL 视图或 Node.js 聚合函数）负责将这些原始数据计算成“当前熟练度分布”、“今日待复习列表”等高度浓缩的 JSON 格式（即 World State）。
* **脚本的自进化：** 当用户要求增加新功能（例如：“帮我加一个分析我哪些词根容易记错的功能”），Developer Agent 不仅会写 UI，还会**主动修改这个数据提取脚本**。它会在脚本层新增针对词根数据的统计 SQL，从而将“高频错词词根列表”动态打包进下一次生成的 World State 中。
* **数据流向闭环：** `Raw DB (海量日志) -> 自进化提取脚本 (按需聚合) -> World State (精简 JSON) -> Agent Context`。

### 2.4 Tools 实现 (原生扩展与安全控制)
* **文件读写与重构 Tools：** 提供原生文件系统读写能力，让 Agent 能够直接重写自身的源代码（包括数据提取层的脚本）。
* **知识检索 RAG 技能：** 接入 `@upstash/vector` 赋予 Agent 实时查阅开发文档或英文长文上下文的能力。
* **硬核本地命令行 Tools：** 提供运行任意 Shell 命令的工具。必须强制设置 `timeout` 并截断 `stdout`。Agent 修改的代码直接映射到本地 Git 开发目录中，方便接管和审查。

## 3. 具体技术栈与落地路径

### 3.1 核心技术栈推荐
* **前端与中间件：** Next.js (App Router) 
* **样式与UI组件：** Tailwind CSS + shadcn/ui 
* **FSRS 算法引擎：** `fsrs.js` 开源库
* **向量数据库：** `@upstash/vector` 
* **持久化存储：** SQLite（本地轻量）或 PostgreSQL。

### 3.2 渐进式进化路线
#### 第一步：5 分钟搭建极速 MVP
快速跑起 Vercel AI SDK 官方模板，接入最强代码大模型，赋予 Agent 执行 Node/Shell 脚本的 Tool。

#### 第二步：打通 UI 自进化闭环
让 Agent 自己写出包含 FSRS 按钮的单词卡片组件，并保存到代码库动态渲染。

#### 第三步：打通数据提取层自进化（Data Pipeline Evolution）
向 Agent 下达指令：
> “帮我写一个统计本周‘易混淆单词’的数据脚本，把它挂载到 World State 的提取管线中。下次我问你我的复习情况时，你需要能看到这份聚合后的统计数据。”

当 Agent 成功修改了后端的聚合脚本，并在后续对话中精准读取到该维度的统计信息时，系统完成了从 UI 到数据底层真正的全面“自进化”。
